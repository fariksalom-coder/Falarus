# Последовательный доступ к урокам и заданиям

## Порог успеха

- **≥ 70%** правильных ответов в задании → задание считается **пройденным**, открывается **следующее** задание.
- **&lt; 70%** → задание можно **пересдать**; следующее задание **не открывается**.

## Уроки

- **Урок 1** всегда открыт первым (после учёта подписки / freemium).
- **Следующий урок** открывается, когда **все** задания текущего урока пройдены с ≥ 70%.

## Данные

- Локально: `localStorage` (`lessonTaskResults`).
- Сервер: таблица `lesson_task_results` (синхронизация через `saveLessonTaskResult` / загрузка через `fetchLessonTaskResults`).
- Контекст `SequentialLessonProvider` объединяет локальные и серверные результаты.

## Код

| Файл | Назначение |
|------|------------|
| `src/lib/sequentialLessonProgress.ts` | Чистая логика: разблокировка урока/задания, завершение урока |
| `src/lib/lessonUrlTaskMapping.ts` | URL → номер задания (в т.ч. особый случай `/lesson-11`) |
| `src/context/SequentialLessonContext.tsx` | Состояние и API `isLessonAccessible` / `isTaskAccessible` |
| `src/components/SequentialAccessEnforcer.tsx` | Редирект при прямом заходе на закрытый маршрут |
| `src/hooks/useLessonsSubscriptionLock.ts` | Блокировка по тарифу (как на дашборде) |

## UI

- **Dashboard**: карточки с учётом подписки + последовательности; галочка при завершении урока.
- Кнопки заданий на страницах уроков по-прежнему ведут на маршруты; **недоступные** URL перехватываются `SequentialAccessEnforcer`.

## Доработка страниц уроков

Чтобы кнопка не вела на заблокированное задание без «мигания» редиректа, на странице можно вызвать `useSequentialLesson().isTaskAccessible(lessonPath, taskNumber)` и отключать кнопку.
